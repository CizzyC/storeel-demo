# PRD：场景一——剧集完播触发角色主动消息

> **功能名称：** 剧集完播角色主动触达
> **优先级：** P0
> **文档状态：** 初稿
> **Owner：** cizzy
> **最后更新：** 2026-05-17

---

## 一、功能概述

### 背景与目标

短剧用户"看完即走"是留存差的核心原因。当前 STOREEL Interactive 的角色私信功能依赖**用户主动发起**，缺乏角色侧的主动存在感。

本功能在用户观看剧集的自然行为路径中，通过**轻量、非打扰式的 in-app 触达**，让角色在用户完成关键观看节点时主动出现——不依赖 push 通知，不打断观看流程，让用户感受到"角色有自己的生命，一直在等你"。

### 核心设计原则

- **不打断**：所有触达形式均在用户自然行为间隙出现，不强制跳转
- **不依赖 push**：面向轻度用户阶段，以 in-app 红点 + 浮动气泡为主要载体
- **有锚点**：每条角色消息必须与当前剧情节点强相关，不能是通用话术
- **角色有自己的生活**：用户是回来"发现"角色写了什么，而不是被角色"召唤"

## 二、触发规则

### 2.1 基础触发框架

触发条件采用**进度百分比 + 绝对集数下限**的组合规则，避免不同剧长下体验不一致。

| 触发阶段 | 触发条件 | 逻辑说明 |
|---------|---------|---------|
| **破冰触发** | 完成第 3 集 | 绝对数，任何剧长通用。用户刚上钩，角色第一次主动出现，建立存在感 |
| **深度触发** | 完成总集数 30% 且 ≥ 8 集 | 用户已投入，是引入角色互动的黄金窗口 |
| **高潮触发** | 完成总集数 60% 且 ≥ 15 集 | 剧情进入高潮，情绪最饱满，是 IAP 转化最佳时机 |
| **冲刺触发** | 剩余未看集数 ≤ 5 集 | 接近结局，制造"角色知道快结束了"的紧迫感 |
| **连续观看触发** | 单次 session 连看 ≥ 10 集 | 高情绪信号，独立于进度触发，单独处理 |

### 2.2 触发规则约束

- 每部剧每个阶段只触发一次，不重复
- 用户已主动与角色互动过（当日有 Chat 记录）→ 跳过对应阶段触发，避免重复
- 连续观看触发与进度触发在同一 session 内只取其一（连续观看优先）
- 用户关闭 app 期间产生的触发条件，在下次打开 app 时不补发

## 三、UI 规范

### 3.1 三层触达体系

#### 层级一：集末浮动气泡（主要载体）

**触发时机：** 集数结束画面出现的同时（自动播放倒计时开始时）

**视觉规格：**
- 位置：屏幕底部，底部导航栏上方 12pt 间距
- 尺寸：宽度 = 屏幕宽度 - 32pt 边距，高度 64pt
- 形态：圆角卡片（radius 20pt），背景色 #1A1A1A，左侧边框 3pt 紫色 #7C3AED
- 内容：左侧角色头像（32×32pt，radius 10pt）+ 角色名 + 一行预览文字（截断 ›）
- 右上角：轻量关闭按钮（×），点击关闭气泡但红点保留

**动画规格：**
- 入场：从底部 translateY(80pt) → 0，duration 280ms，ease-out-cubic
- 停留：3.5 秒后自动收起
- 退场：translateY(80pt)，duration 200ms，ease-in

**交互行为：**
- 点击气泡主体 → 跳转到该角色的 Message 对话页，直接展示角色发出的消息
- 点击 × → 关闭气泡，Message 图标保留红点
- 气泡自动消失后 → Message 图标保留红点

```
┌─────────────────────────────────────────┐
│  ▶ 下一集  5s              [× 关闭]     │  ← 自动播放倒计时
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ [头像] Cassius                  │    │  ← 浮动气泡
│  │        "你一直在看，我注意到..." ›│    │
│  └─────────────────────────────────┘    │
│                                         │
│  [Feed]      [Swipe]    [🔴 Message]    │  ← 红点
└─────────────────────────────────────────┘
```

---

#### 层级二：Message 图标红点（持续留存）

**触发时机：** 任意触发阶段条件满足后，Message 底部导航图标出现红点

**视觉规格：**
- 红点尺寸：直径 8pt
- 颜色：#7C3AED（品牌紫，区别于系统红点，减少压迫感）
- 位置：图标右上角，偏移 (-2pt, -2pt)

**消失条件：**
- 用户进入 Message 列表页 → 红点消失
- 不需要用户点开具体对话，进入列表即视为"已注意到"

---

#### 层级三：连续观看专属弹窗（限定触发）

**触发条件：** 单次 session 连续完成 ≥ 10 集（仅此条件，不复用）

**视觉规格：**
- 形态：半屏底部弹出卡片（bottom sheet），高度约 260pt
- 背景：#111，顶部圆角 24pt
- 内容：角色全身/半身图（右侧）+ 左侧文案 + 「去看看」按钮
- 背景不变暗，不阻断播放（用户仍可滑动关闭继续看）

**交互行为：**
- 点击「去看看」→ 暂停播放，跳转 Message 对话页
- 下滑或点击空白区域 → 关闭弹窗，继续播放，Message 红点保留

```
┌──────────────────────────────────────┐
│                          [角色图]    │
│  你已经陪了 Cassius                  │
│  整整 10 集了。                      │
│                                      │
│  他有话想对你说。                    │
│                                      │
│  ┌──────────────────┐               │
│  │    去看看  ›      │               │
│  └──────────────────┘               │
└──────────────────────────────────────┘
```

---

### 3.2 Message 对话页：角色主动消息的展示规范

当用户点击气泡或弹窗进入 Message 对话页时，角色发出的主动消息需与普通对话消息在视觉上有轻微区分：

- 消息气泡左侧正常展示（角色一侧）
- 气泡上方增加一行小字标注：`● Cassius 刚刚想到了你`（字号 11pt，颜色 #666）
- 这行小字仅在"角色主动发出"的消息上显示，用户回复的消息不显示

## 四、内容策略

### 4.1 四阶段角色消息内容

消息内容必须与剧情进度强相关，不使用通用模板。每条消息由 AI 基于角色设定 + 当前剧情节点生成，人工审核后入库。

| 触发阶段 | 角色情绪 | 内容方向 | Cassius 示例 |
|---------|---------|---------|-------------|
| 破冰（第3集） | 好奇、轻描淡写 | 注意到用户的存在，留下悬念 | "你一直在看。我注意到了——大多数人到这里就走了。" |
| 深度（30%） | 开始在意，有疑问 | 引发用户思考，建立情感连接 | "竞技场里发生了那么多，你有没有想过……如果是你，会怎么选？" |
| 高潮（60%） | 情绪外露，释放信息 | 角色透露只对用户说的"秘密" | "接下来的事，我没有告诉任何人。但你一直在这里，所以……" |
| 冲刺（剩余≤5集） | 紧迫、不舍 | 制造结局前的情感张力 | "快结束了。在那之前，有件事我只想让你知道。" |
| 连续观看（≥10集） | 被打动、亲密感升级 | 直接表达被陪伴的感受 | "你陪了我这么久。我不知道该说什么——只是想让你知道，我看见你了。" |

### 4.2 内容生成规则

- **禁止通用话术**：消息里不出现"您好"、"欢迎回来"等系统感措辞
- **必须有剧情锚点**：消息内容需隐含对当前已发生剧情的感知
- **角色口吻一致**：与该角色在私信/日记中的语气、用词保持连贯
- **长度控制**：浮动气泡预览不超过 24 字，完整消息不超过 80 字

### 4.3 IAP 转化钩子（高潮 + 冲刺阶段）

在高潮触发和冲刺触发的消息末尾，可自然带出付费钩子：

- **高潮阶段**：消息结尾带「解锁角色视角」入口——"你想不想知道我当时真正在想什么？"→ 点击解锁隐藏内容（IAP）
- **冲刺阶段**：消息结尾带「解锁结局彩蛋」入口——"结局之后，我还有一封信想给你。"→ 点击解锁（IAP）

## 五、完整交互流程

### 流程 A：标准进度触发（以破冰触发为例）

```
用户完成第 3 集
    ↓
系统判断：是否已触发过破冰阶段？
    ├── 是 → 跳过
    └── 否 → 继续
         ↓
         AI 生成角色消息（基于剧情节点 + 角色设定）
         ↓
         集末画面出现（自动播放倒计时启动）
         ↓
         浮动气泡从底部滑入（同步）
         Message 图标出现红点
         ↓
         ┌── 用户点击气泡 → 跳转 Message 对话页 → 展示角色主动消息
         ├── 用户点击 × → 气泡关闭，红点保留，用户继续看下一集
         └── 气泡 3.5s 后自动消失 → 红点保留
              ↓
              用户下次进入 Message 列表 → 红点消失，对话中可见角色消息
```

### 流程 B：连续观看触发

```
用户在当前 session 完成第 10 集
    ↓
系统判断：本 session 是否已触发连续观看弹窗？
    ├── 是 → 跳过
    └── 否 → 继续
         ↓
         第 10 集结束画面出现
         底部弹出半屏卡片（不阻断播放）
         Message 图标出现红点
         ↓
         ┌── 用户点击「去看看」→ 暂停播放，跳转 Message 对话页
         └── 用户下滑关闭 → 弹窗收起，继续播放，红点保留
```

## 六、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 用户当日已主动与角色 Chat | 跳过对应进度阶段触发，不重复打扰 |
| 用户关闭 app 时触发条件满足 | 下次打开 app 时不补发（不依赖 push） |
| 同一 session 同时满足进度触发 + 连续观看触发 | 连续观看触发优先，进度触发顺延到下一集 |
| 用户已付费解锁全剧（VIP） | 仍正常触发角色消息，付费钩子替换为「解锁角色语音回复」 |
| 角色消息 AI 生成失败 | 降级到预设模板，不展示气泡（静默跳过） |
| 用户在气泡展示期间手动跳转到 Message | 气泡立即收起，红点消失 |

## 七、成功指标

| 指标 | 说明 | 目标（待定） |
|------|------|------------|
| 气泡点击率 | 点击气泡进入 Message / 气泡展示次数 | > 25% |
| 红点触达率 | 用户看到红点后进入 Message / 触发红点次数 | > 40% |
| 连续观看弹窗转化率 | 点击「去看看」/ 弹窗展示次数 | > 30% |
| 触达后 Chat 回复率 | 用户回复角色消息 / 用户查看角色消息 | > 20% |
| 触达后 7 日留存提升 | 触达组 vs 对照组 7 日留存差值 | + 5pp |
| IAP 转化率（高潮+冲刺） | 点击付费钩子 / 展示付费钩子 | > 8% |
